# Генерация CV под вакансию

## Что делает этот скилл

Берёт резюме из cv.md, адаптирует формулировки под конкретную вакансию, генерирует HTML из шаблона и конвертирует в PDF.

## Как использовать

`/cv-gen 12345678` (ID вакансии) или `/cv-gen https://hh.ru/vacancy/12345678`

## Инструкции

### Шаг 1: Подготовка

1. Прочитай `cv.md` — весь опыт, навыки, образование
2. Прочитай `config/profile.yml` — контакты, позиционирование
3. Получи описание вакансии:
   ```bash
   curl -s -H "User-Agent: HHCareerOps/1.0" "https://api.hh.ru/vacancies/{ID}"
   ```
4. Если есть отчёт оценки в `reports/` для этой вакансии — прочитай блок E (Персонализация)

### Шаг 2: Извлечение ключевых слов

Из описания вакансии извлеки 15-20 ключевых слов/фраз:
- Технологии и инструменты (Make.com, CRM, ChatGPT и т.д.)
- Навыки (автоматизация, лидогенерация, аналитика)
- Бизнес-термины (unit-экономика, воронка, конверсия)

### Шаг 3: Адаптация контента

**ПРАВИЛО: только переформулировать, НИКОГДА не придумывать**

1. **Summary** — перепиши 2-3 предложения, вставив 3-5 ключевых слов из вакансии
2. **Опыт** — для каждой позиции из cv.md:
   - Переставь bullet points: самые релевантные для данной вакансии — наверх
   - Переформулируй, используя терминологию из вакансии
   - Сохрани все цифры и метрики как есть
3. **Навыки** — добавь навыки из cv.md, которые упоминаются в вакансии, в первые позиции

### Шаг 4: Генерация HTML

Прочитай `templates/cv-template.html` и замени токены:

- `{{NAME}}` — имя из profile.yml
- `{{CONTACTS}}` — email, телефон, telegram, linkedin через " | "
- `{{SUMMARY}}` — адаптированное summary
- `{{EXPERIENCE}}` — блоки опыта в формате:
  ```html
  <div class="job">
    <div class="job-header">
      <span class="job-title">Должность</span>
      <span class="job-dates">Мес Год — Мес Год</span>
    </div>
    <div class="job-company">Компания</div>
    <ul>
      <li>Достижение 1</li>
      <li>Достижение 2</li>
    </ul>
  </div>
  ```
- `{{SKILLS}}` — теги навыков:
  ```html
  <span class="skill-tag">Навык</span>
  ```
- `{{EDUCATION}}` — образование:
  ```html
  <div class="education-item">
    <strong>Университет</strong> — Специальность, Год
  </div>
  ```
- `{{ADDITIONAL}}` — дополнительные секции (сертификаты, проекты) или пустая строка

### Шаг 5: Сохранение и генерация

1. Определи slug компании из названия (латиницей, через дефис)
2. Сохрани HTML: `output/cv-{company-slug}-{YYYY-MM-DD}.html`
3. Запусти:
   ```bash
   node scripts/generate-pdf.mjs "output/cv-{company-slug}-{YYYY-MM-DD}.html" "output/cv-{company-slug}-{YYYY-MM-DD}.pdf"
   ```
4. Если PDF успешно создан, обнови трекер: колонка CV = ссылка на PDF

### Шаг 6: Вывод

Сообщи пользователю:
- PDF создан: `output/cv-{...}.pdf`
- Ключевые изменения относительно базового CV
- Количество ключевых слов из вакансии, встроенных в CV
- Предложи: "Откликнуться на эту вакансию? (`/apply {HH_ID}`)"

## Важные правила
- НИКОГДА не придумывай опыт, навыки или метрики
- Только переформулируй и переставляй реальное из cv.md
- PDF должен быть одностраничным (если возможно), максимум 2 страницы
- Текст должен быть выделяемым (не картинка)
- Шрифты — стандартные, без экзотики
- Формат A4
