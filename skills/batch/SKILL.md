# Пакетная обработка вакансий

## Что делает этот скилл

Запускает полный цикл: поиск новых вакансий → фильтрация → оценка каждой → генерация CV для лучших → итоговый отчёт.

## Как использовать

- `/batch` — полный цикл по всем запросам
- `/batch топ` — только оценка + CV для оценок 4.0+
- `/batch поиск` — только поиск без оценки

## Инструкции

### Режим "полный цикл" (`/batch`)

1. **Поиск:**
   ```bash
   node scripts/search-hh.mjs --all
   ```
   Прочитай результаты из `data/last-search.json`

2. **Фильтрация:**
   - Убери дубликаты (уже есть в трекере по HH ID)
   - Покажи пользователю список найденных вакансий
   - Спроси: "Оценить все {N} вакансий? Или выбери номера."

3. **Оценка:**
   Для каждой выбранной вакансии выполни скилл `/evaluate`:
   - Получи описание через API
   - Проведи 6-блочную оценку
   - Сохрани отчёт
   - Добавь в трекер
   - Используй sub-agents для параллельной оценки (до 3 одновременно)

4. **Генерация CV:**
   Для вакансий с оценкой >= 4.0:
   - Спроси пользователя: "Сгенерировать CV для {N} вакансий с оценкой 4.0+?"
   - После подтверждения — выполни `/cv-gen` для каждой

5. **Итоговый отчёт:**
   ```
   === Пакетная обработка завершена ===
   
   Найдено новых: N
   Оценено: N
   
   Рекомендовано (4.0+): N
   - {Компания} — {Роль} (оценка X.X)
   - {Компания} — {Роль} (оценка X.X)
   
   Можно откликнуться (4.5+): N
   CV сгенерированы: N
   
   Пропущено (<3.5): N
   
   Следующий шаг: /apply {ID} для отклика
   ```

### Режим "только поиск" (`/batch поиск`)

Только шаги 1-2: поиск + фильтрация + вывод списка.

### Режим "топ" (`/batch топ`)

Фильтрует трекер по оценкам 4.0+ без CV, генерирует CV для них.

## Правила
- Между запросами к API — пауза 300мс (встроена в search-hh.mjs)
- Не откликаться автоматически — только оценка и CV
- Если API не отвечает — продолжить с остальными запросами
- Максимум 50 вакансий за один батч (чтобы не перегрузить)
