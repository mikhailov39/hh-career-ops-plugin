# Первоначальная настройка HH Career Ops

## Что делает этот скилл

Помогает настроить систему при первом запуске на новом компьютере.

## Как использовать

`/setup`

## Инструкции

### Шаг 1: Проверка зависимостей

1. Проверь что Node.js установлен:
   ```bash
   node --version
   ```
2. Установи зависимости:
   ```bash
   npm install
   ```

### Шаг 2: Создание config/profile.yml

1. Проверь есть ли `config/profile.yml`
2. Если нет — скопируй из шаблона:
   ```bash
   cp config/profile.example.yml config/profile.yml
   ```
3. Спроси пользователя:
   - Имя, email, телефон, город
   - Целевые роли (из 3 уровней: primary, secondary, adjacent)
   - Зарплатные ожидания
   - Формула позиционирования
4. Заполни profile.yml

### Шаг 3: Создание cv.md

1. Проверь есть ли `cv.md`
2. Если нет — скопируй из шаблона:
   ```bash
   cp cv.example.md cv.md
   ```
3. Попроси пользователя заполнить или прислать PDF-резюме
4. Если прислал PDF — извлеки текст и заполни cv.md

### Шаг 4: Проверка API hh.ru

```bash
node scripts/search-hh.mjs --query "test" --limit 1
```

Должен вернуть JSON с вакансиями. Если ошибка — проверь интернет.

### Шаг 5: Настройка AdsPower (опционально)

1. Проверь доступность AdsPower:
   - Используй `check-status`
2. Если работает — спроси пользователя какой профиль использовать для hh.ru
3. Запиши profile_id в config/profile.yml
4. Проверь что профиль открывается и hh.ru загружается

### Шаг 6: Настройка queries.yml (опционально)

Спроси пользователя:
- Хочет ли изменить поисковые запросы
- Нужно ли добавить/убрать фильтры
- Правильный ли город и зарплата

### Шаг 7: Итог

Выведи:
```
✅ HH Career Ops настроен!

Профиль: {имя} — {город}, от {зп}к
Резюме: cv.md заполнено
API hh.ru: работает
AdsPower: {подключен/не настроен}

Команды:
/hh-search — поиск вакансий
/evaluate <URL> — оценка вакансии
/cv-gen <ID> — генерация CV
/apply <ID> — отклик через AdsPower
/batch — полный цикл
/tracker — просмотр трекера
```
