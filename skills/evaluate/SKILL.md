# Оценка вакансии с hh.ru

## Что делает этот скилл

Принимает URL или ID вакансии с hh.ru, получает полное описание, проводит 6-блочную оценку, выставляет скоринг 1-5, сохраняет отчёт и добавляет в трекер.

## Как использовать

`/evaluate https://hh.ru/vacancy/12345678` или `/evaluate 12345678`

## Инструкции

### Шаг 1: Подготовка

1. Прочитай `cv.md` — это единственный источник правды об опыте кандидата
2. Прочитай `config/profile.yml` — целевые роли, позиционирование, ЗП
3. Извлеки ID вакансии из URL (число после `/vacancy/`)

### Шаг 2: Получение вакансии

Запусти:
```bash
curl -s -H "User-Agent: HHCareerOps/1.0" "https://api.hh.ru/vacancies/{ID}"
```

Или используй WebFetch на URL вакансии.

### Шаг 3: 6-блочная оценка

#### Блок A: Сводка роли
| Поле | Значение |
|------|----------|
| Компания | из employer.name |
| Должность | из name |
| ЗП | из salary (from-to, gross/net) |
| Формат | remote/hybrid/office из work_format |
| Опыт | из experience |
| Город | из area.name |
| Аккредитованный IT | из employer.accredited_it_employer |

Одно предложение TL;DR: что за роль и зачем она компании.

#### Блок B: Совпадение с CV (вес: 30%)
- Выпиши каждое требование из вакансии
- Напротив каждого — конкретная строка из cv.md, которая подтверждает навык
- Если нет подтверждения — пометь как "ГЭП"
- Посчитай % покрытия требований
- Хард-блокеры (must-have без покрытия) отмечай отдельно

Оценка:
- 5: покрытие 90%+, нет хард-блокеров
- 4: покрытие 70-89%, блокеры закрываемы
- 3: покрытие 50-69%
- 2: покрытие 30-49%
- 1: покрытие <30% или критические блокеры

#### Блок C: Позиционирование (вес: 20%)
Как продать профиль "AI + бизнес-автоматизация" под эту вакансию:
- Какой архетип из profile.yml лучше подходит
- Формула позиционирования под конкретную роль
- Какие достижения из cv.md подсветить в первую очередь
- Риски (overqualified? underqualified? не тот профиль?)

Оценка:
- 5: идеальное совпадение с целевым профилем
- 4: хорошее совпадение, минимальная адаптация
- 3: нужна значительная адаптация позиционирования
- 2: роль далека от целевого профиля
- 1: совсем не тот профиль

#### Блок D: Компенсация и рынок (вес: 20%)
- Указанная ЗП vs ожидания из profile.yml
- Если ЗП не указана — оценка по рынку для данной роли в Москве
- Аккредитованный IT-работодатель = плюс (льготная ипотека, отсрочка)

Оценка:
- 5: выше ожиданий (>400к) или рыночный топ
- 4: в целевом диапазоне (200-400к)
- 3: на нижней границе (~200к)
- 2: ниже ожиданий (<200к)
- 1: значительно ниже рынка

#### Блок E: Персонализация CV (вес: 20%)
- Топ-5 ключевых слов из вакансии для вставки в CV
- Какие bullet points из cv.md переформулировать (НЕ придумывать новые)
- Рекомендация по summary/заголовку
- Рекомендация по сопроводительному письму (3-4 предложения)

Оценка: не скорится, это план действий.

#### Блок F: Легитимность (вес: 10%)
Сигналы:
- Возраст вакансии (из published_at): <30 дней = ОК, 30-60 = осторожно, >60 = подозрительно
- Детальность описания: конкретные задачи vs общие слова
- response_letter_required = серьёзный работодатель
- has_test = дополнительный этап
- employer.trusted = проверенный работодатель

Оценка:
- 5: свежая, детальная, проверенный работодатель
- 4: нормальная вакансия
- 3: есть сомнения
- 2: подозрительная
- 1: скорее всего неактуальна или мошенничество

### Шаг 4: Итоговый скоринг

**Формула:** `(B × 0.30) + (C × 0.20) + (D × 0.20) + (F × 0.10) + бонус`

Бонус +0.2 за:
- Аккредитованный IT-работодатель
- Удалённый формат
- Целевая роль из primary в profile.yml

**Пороги:**
- **4.5+** — откликаться сразу, генерировать CV
- **4.0–4.4** — стоит откликнуться
- **3.5–3.9** — только если есть причина
- **<3.5** — пропустить (SKIP)

### Шаг 5: Сохранение отчёта

Определи порядковый номер: посчитай строки в `data/tracker.md` (минус заголовок).

Сохрани отчёт в `reports/{NNN}-{company-slug}-{YYYY-MM-DD}.md`:
```markdown
# Оценка: {Компания} — {Должность}
**URL:** {alternate_url}
**Дата оценки:** {сегодня}
**Итоговая оценка: {X.X}/5**

## A: Сводка роли
[таблица]

## B: Совпадение с CV ({X}/5)
[маппинг требований]

## C: Позиционирование ({X}/5)
[анализ]

## D: Компенсация ({X}/5)
[анализ]

## E: Персонализация CV
[план]

## F: Легитимность ({X}/5)
[сигналы]

## Рекомендация
[Откликаться / Мониторить / Пропустить]

## Сопроводительное письмо (черновик)
[3-4 предложения]
```

### Шаг 6: Обновление трекера

Добавь строку в `data/tracker.md`:
```
| {NNN} | {YYYY-MM-DD} | {Компания} | {Должность} | {ЗП} | {X.X}/5 | Оценена | — | [отчёт](../reports/{file}) | {HH_ID} |
```

### Шаг 7: Рекомендация

Сообщи пользователю:
- Итоговую оценку и рекомендацию
- Топ-3 плюса и топ-3 риска
- Если оценка >= 4.0, предложи: "Сгенерировать CV под эту вакансию? (`/cv-gen {HH_ID}`)"

## Важные правила
- НИКОГДА не придумывай опыт — только переформулируй реальное из cv.md
- ВСЕГДА читай cv.md и profile.yml перед оценкой (каждый раз)
- Используй русский язык в отчётах
- HH ID — это число из URL вакансии
