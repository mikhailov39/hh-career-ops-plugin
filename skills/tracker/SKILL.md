# Трекер вакансий

## Что делает этот скилл

Управляет файлом `data/tracker.md` — показывает вакансии, фильтрует по статусу, обновляет статусы, выводит статистику.

## Как использовать

- `/tracker` — показать все вакансии
- `/tracker статистика` — сводка по статусам
- `/tracker отклик` — показать только со статусом "Отклик"
- `/tracker обновить 5 Собеседование` — обновить статус вакансии #5

## Инструкции

### Просмотр

1. Прочитай `data/tracker.md`
2. Выведи таблицу пользователю
3. Если указан фильтр по статусу — покажи только подходящие строки

### Статистика

Посчитай и выведи:
```
Всего вакансий: N
- Оценена: N
- Отклик: N
- Ответ: N
- Собеседование: N
- Оффер: N
- Отказ: N
- Пропуск: N

Средняя оценка: X.X/5
Топ-3 по оценке: [список]
```

### Обновление статуса

Канонические статусы:
| Статус | Значение |
|--------|----------|
| Оценена | Оценка проведена, решение не принято |
| Отклик | Отклик отправлен |
| Ответ | Компания ответила |
| Собеседование | Назначено/проходит собеседование |
| Оффер | Получен оффер |
| Отказ | Отказ от компании |
| Пропуск | Решено не откликаться |

При обновлении:
1. Найди строку по номеру (#) в трекере
2. Замени значение в колонке "Статус"
3. Сохрани файл
4. Подтверди пользователю: "Вакансия #{N} ({Компания} — {Роль}): статус обновлён на {Статус}"

### Добавление вручную

Если пользователь хочет добавить вакансию вручную (не через /evaluate):
1. Определи следующий порядковый номер
2. Добавь строку с минимальными данными
3. Предложи провести оценку: `/evaluate {URL}`

### Дедупликация

Перед добавлением проверяй HH ID — если уже есть в трекере, сообщи пользователю и не добавляй дубликат.

## Правила
- Трекер — единственное место учёта, не дублируй данные
- Не удаляй строки — только меняй статусы
- Сортировка: по номеру (хронологическая)
